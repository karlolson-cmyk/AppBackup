## Appkitz v2.9.0 Release（优化体积）

### 问题
- v2.8.0 关闭混淆后 APK 体积 26.6MB，用户要求压缩到 15MB 以内

### 解决方案
1. **重新启用 R8 混淆** (`isMinifyEnabled = true`, `isShrinkResources = true`)
2. **精简 ProGuard 规则**: 不手动 keep 整个库包，让库自带的 consumer rules 处理
   - 只 keep `com.appkitz.**`（应用代码）和 `com.reandroid.**`（ARSCLib，无 consumer rules）
   - 对 OkHttp/OkIO/ErrorProne 只用 `-dontwarn`，不用 `-keep`
   - Compose/AndroidX 自带 consumer ProGuard rules，无需手动 keep
3. **补充缺失类规则**: `com.google.errorprone.annotations.**` 的 `-dontwarn`（Tink/SecurityCrypto 引用）
4. **结果**: APK 6.9MB（从 26.6MB 降至 6.9MB，降幅 74%）

### 关键经验
- 现代 Android 库（Compose, OkHttp, AndroidX 等）自带 consumer ProGuard rules，不需要手动 keep
- 手动 `-keep class xxx.** { *; }` 会阻止 R8 裁剪，导致体积膨胀
- 只需 keep 无 consumer rules 的库（如 ARSCLib）和应用自身代码
- R8 missing_rules.txt 会提示需要添加的 `-dontwarn` 规则
